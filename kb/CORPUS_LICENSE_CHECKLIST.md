# KBコーパス著作権チェックリスト

`kb/corpus/` に新規ファイルを追加する、または既存ファイルを更新する際は、
コミット前に以下の項目をすべて確認すること。本チェックリストは
要件定義書 §11(著作権・ライセンス方針)への準拠を目的とする。

## 必須確認項目

- [ ] **原文の複製・逐語再現がないこと**: 本文は執筆者(またはAI支援の
      下で執筆者が編集・承認した)独自の要約・解説であり、PMBOK®ガイド
      その他の著作物からの文章の複製・ほぼ逐語的な言い換えを含まない。
      引用が必要な場合も、要点を独自の言葉で再構成すること。
- [ ] **商標の非製品名的用法**: 「PMBOK」「PMP」等の商標文字列を
      製品名・タイトル・見出しとして使用していない。参考文献としての
      言及であっても、コーパス本文中には含めない方針とする
      (脚注・出典管理は別ドキュメントで行う)。
- [ ] **出典の記録**: front-matterの `source` フィールドに出典種別
      (`original`: 独自著作、またはライセンス上再配布可能な資料からの
      派生である場合はその出典)を明記している。
- [ ] **ライセンスの明記**: front-matterの `license` フィールドに
      本ファイルの利用条件(既定は `internal`: 社内利用限定)を明記
      している。
- [ ] **必須メタデータの充足**: `domain`, `title`, `source`, `license`
      の4フィールドがすべて存在し、`domain` は既知の値
      (`kb_ingest.frontmatter.KNOWN_DOMAINS`)のいずれかである。
- [ ] **フレームワーク名の一般名詞としての使用**: `framework`
      (RICE/WSJF/Kano/AARRR等)や `pm_principle`
      (focus_on_value/tailoring等)は、特定書籍の見出し構成そのものの
      複製ではなく、一般に流通する概念名として言及している。

## レビュー手順

1. 執筆者は本チェックリストの全項目を確認した上でPRを作成する。
2. レビュアーは特に「原文の複製・逐語再現がないこと」を重点的に確認する
   (类似の言い回しが複数箇所にある場合は原典との突き合わせを行う)。
3. `kb/tests/test_corpus_frontmatter.py` のCIチェック
   (front-matterスキーマ検証、商標文字列の不使用確認)がグリーンで
   あることを確認する。

## 参考

- 要件定義書 §11: PMBOK®ガイド原文はPMIの著作物であり、原文の複製・
  学習投入・逐語再現は行わない。隣接領域KBは独自著作の要約・解説のみで
  構成する。「PMBOK」「PMP」等の商標は製品名に使用しない。
