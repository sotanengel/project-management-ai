# 運用規程 (責任分界)

本ドキュメントは要件定義書 §11 の責任分界を Tier-L 運用向けに明文化したものです。

## 1. 基本方針

- エージェントの起案・実行はすべて監査ログと PMDF `provenance` で
  人間の承認者に紐づけます。
- 自律レベル (L0〜L3) はプロダクト×業務種別ごとに管理者が設定します。
- 緊急停止は管理者のみが操作でき、全エージェント自律実行を即時停止します。

## 2. 自律レベルと責任

| レベル | 意味                                | 最終責任者                                                  |
| ------ | ----------------------------------- | ----------------------------------------------------------- |
| L0     | 手動のみ                            | 操作したユーザー                                            |
| L1     | 起案は自動、**実行は承認必須**      | **承認した人間** (FR-PD ビジョン・ロードマップ・リリース等) |
| L2     | 起案・実行とも即時反映 (記録は残る) | 自律レベルを設定した**管理者**                              |
| L3     | 記録中心 (Decision Record 等)       | 自律レベルを設定した**管理者**                              |

要件引用:

> L1 業務の最終責任は承認した人間に、L2/L3 業務の運用責任は自律レベルを
> 設定した管理者に帰属する。

## 3. 承認フロー (L1)

1. エージェントまたは編集者が起案 (`POST /approvals`)
2. 承認者が監督 UI の承認キューで承認/差し戻し
3. 承認後のみ L1 実行系 API (例: `/roadmap/{id}/confirm`) が成功する
4. 承認記録は PMDF `approval` エンティティとして pmdf-store に永続化

差し戻し時は理由が必須です (API 422 + UI バリデーション)。

## 4. 予算・学習

- 月次予算の 80% で `budget.warning` イベントを発行します。
- 100% 超過時は学習ループを自動停止し、`data/budget_exceeded.json` に記録します。
- 予算設定の変更は管理者権限とします。

## 5. バックアップ

- `scripts/backup.sh` で pmdf-store / kb / 学習データを取得します。
- `.env` 等のシークレットは**バックアップに含めません** (SEC-01)。
- リストアは `scripts/restore.sh` を Git Bash / WSL で実行します。

## 6. セキュリティ

- 週次の `security-scan.yml` (Trivy / pip-audit / pnpm audit) を確認します。
- 依存更新は CI 結果を踏まえ issue で追跡します。

## 7. インシデント対応

1. 緊急停止 (監督 UI)
2. 影響範囲の特定 (監査ログ `/audit/records`、活動ログ)
3. pmdf-store の版巻き戻し (Git 履歴) またはバックアップからのリストア
4. 事後に Decision Record / 週次レビューで記録
